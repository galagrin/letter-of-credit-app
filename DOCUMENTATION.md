## LC-App: Система Управления Аккредитивами

LC-App — это full-stack веб-приложение, предназначенное для учета и управления документарными аккредитивами. Проект построен на современном стеке технологий, включая Next.js, TypeScript, PostgreSQL и Prisma, и демонстрирует реализацию сложной бизнес-логики, аутентификации, ролевой модели и интерактивного пользовательского интерфейса.

### 🚀 Основные Возможности (Features)

Аутентификация и Авторизация:

-   Регистрация новых пользователей.
-   Вход в систему по email и паролю.
-   Система ролей: ADMIN и USER с разными правами доступа.
-   Защищенные API-эндпоинты и страницы, доступные только авторизованным пользователям.

Управление Сущностями (CRUD):

-   Аккредитивы: Создание, чтение, обновление и удаление (CRUD) аккредитивов.- Пользователи могут редактировать/удалять только свои записи, администраторы — любые.
-   Банки: Полный CRUD для управления списком банков-участников.
-   Компании: Полный CRUD для управления списком компаний (заявителей и бенефициаров).

Интерактивный Пользовательский Интерфейс:

-   Динамический UI, который обновляется в реальном времени без перезагрузки страницы.
-   Модальные окна для создания и редактирования записей.
-   Формы с клиентской валидацией.
-   Табличное представление данных с возможностью управления.

### 🛠️ Технологический Стек

Бэкенд

-   Фреймворк: Next.js (App Router, Route Handlers)
-   Язык: TypeScript
-   База данных: PostgreSQL
-   Окружение БД: Docker и docker-compose для легкого локального развертывания.
-   ORM: Prisma для безопасного и типизированного взаимодействия с базой данных.
-   Аутентификация: NextAuth.js для управления сессиями и учетными данными.
-   Валидация: Zod для строгой валидации данных в API.

Фронтенд

-   Фреймворк: Next.js / React
-   Язык: TypeScript
-   Управление состоянием сервера: TanStack Query (React Query) для кеширования, фонового обновления и мутаций данных.
-   Управление формами: React Hook Form для производительных и управляемых форм.
-   Стилизация: (В процессе) Планируется переход на Tailwind CSS.

### Итоги разработки: Что готово в проекте на данный момент

#### I. Фундамент и Бэкенд (Backend & Infrastructure)

Настроено Окружение:

-   Проект развернут на Next.js с использованием App Router.
-   Настроен TypeScript для строгой типизации всего проекта.
-   Настроена система контроля версий Git.
-   Внедрен и используется менеджер пакетов pnpm для быстрой и надежной установки зависимостей.

Настроена База Данных:

-   Развернута и настроена реляционная база данных PostgreSQL в изолированном Docker-контейнере.
-   Настроена Prisma как ORM для удобного и безопасного взаимодействия с базой данных.
-   Созданы и применены миграции для всех сущностей.

Реализованы Модели Данных:

-   User: Модель пользователя с ролями.
-   Role: enum для ролей (USER, ADMIN).
-   Bank: Модель банка с реквизитами.
-   Company: Модель компании (аппликант/бенефициар).
-   LetterOfCredit: Центральная, самая сложная модель, со всеми связями (многие-ко-многим через разные роли).
-   Currency: enum для валют.

Реализован Полноценный REST API:

-   Аутентификация: Полностью настроена библиотека next-auth. Реализованы эндпоинты для регистрации (POST /api/users), входа (POST /api/auth/callback/credentials), выхода и получения сессии.
-   Авторизация: Внедрена ролевая модель. API-эндпоинты защищены в зависимости от роли пользователя (ADMIN или обычный USER).
-   CRUD для Банков: Реализованы все операции (Создание, Чтение, Обновление, Удаление) через GET, POST, PUT, DELETE на /api/banks.
-   CRUD для Компаний: Реализованы все операции для /api/companies.
-   CRUD для Аккредитивов: Реализованы все операции для /api/lcs, включая сложную логику проверки прав доступа (владелец или админ).
-   Валидация: Все входящие данные на POST и PUT эндпоинтах валидируются с помощью Zod.

#### II. Фронтенд (Frontend & User Interface)

Настроена Архитектура:
Создана базовая структура страниц с помощью App Router (layout.tsx, page.tsx).
Реализован провайдер для next-auth и React Query, обеспечивающий доступ к сессии и кешу данных во всем приложении.

Реализована Аутентификация на UI:

-   Создана кастомная страница входа (/login) с формой и обработкой ошибок.
-   Создан компонент-шапка (Header), который динамически отображает статус пользователя ("Привет, ...!" или кнопка "Войти") и позволяет выйти из системы.

Создан Функциональный Интерфейс:

-   Реализована защита страниц на уровне серверных компонентов (редирект неавторизованных пользователей).
-   Создана Приборная панель (/dashboard) с навигацией, уважающей роли пользователей.
-   Реализованы полноценные CRUD-интерфейсы для управления Банками, Компаниями - Аккредитивами, включая:
    Отображение данных в таблицах.
    Модальные окна для создания и редактирования.
    Формы с валидацией, созданные с помощью react-hook-form.
    Кнопки удаления с подтверждением.- Внедрено Продвинутое Управление Состоянием:
-   Вся логика получения и изменения серверных данных на клиенте полностью переведена на React Query (TanStack Query).
-   Используются хуки useQuery для кеширования и фонового обновления данных.
-   Используются хуки useMutation с invalidateQueries для автоматического обновления интерфейса после создания, изменения или удаления записей.
